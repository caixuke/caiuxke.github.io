手机号验证-发送验证码接口

接口功能

输入手机号之后进行验证，如果已注册直接发送验证码，如果未注册则返回未注册的报错信息

接口参数

  • 测试环境api地址: http://api-test.jinlu-wl.com/
  • Method: GET
  • URL: /oauth/public/send-phone-captcha
  • Params：
    参数名 数据类型 是否必填 说明
    internationalTelCode string N 默认+86
    phone string Y 手机号
    userType string N 用户类型，默认为P
    businessScope string N 商业类型

Response

  • 手机号未注册返回Body
  • 手机号已注册，发送验证码成功body

手机验证码登录

接口功能

输入手机号之后进行验证码登录

接口参数

  • 测试环境api地址: http://api-test.jinlu-wl.com/
  • Method: POST
  • URL: /oauth/token/mobile
  • Params：
    参数名 数据类型 是否必填 说明
    client_id string Y 客户端id，可填client
    client_secret string Y 客户端秘钥，可填secret
    grant_type string Y 授权类型，分implicit和password两种，此处填implicit
    source_type string Y 设备来源，web-Web端/app-移动设备端
    captcha string Y 用户输入的手机验证码
    captchaKey string Y 发送手机验证码返回的captchaKey
    phone string Y 手机号
    device_id string Y 设备ID，可以使用UUID，用于设置 Token 的唯一性
    user_type string N 用户类型：P-平台用户/C-C端用户，默认 P

POSTMAN截图

Response

  • 手机号未验证
  • 手机号未注册 成功登录
